# ADHD-Friendly UI Principles

This document outlines the interaction design standards for the AI Agent Platform, optimized for cognitive accessibility and focus.

## Core Mandate
The UI must feel like a **calm command center**, not a puzzle.

## The Three Questions
Every screen must inherently answer:
1.  **Where am I?** (Clear context & breadcrumbs)
2.  **What matters right now?** (Visual hierarchy of the active task)
3.  **What happens next?** (Predictable primary action)

## Design Guidelines

1.  **Focus Anchors**: Keep primary anchors visible. Avoid jarring context shifts.
2.  **Progressive Disclosure**: Only show complex details when the user explicitly requests them.
3.  **Visual Clarity**: One meaning per visual element. Avoid ambiguous icons without labels.
4.  **Click Targets**: Use large targets. Minimize precision-required movement.
5.  **Modal Discipline**: Avoid interruptive popups ("modal spam"). Prefer inline expansion or dedicated sidebars.
6.  **Dimming over Hiding**: Prefer dimming inactive elements to maintain spatial awareness rather than removing them entirely.
7.  **Cognitive Recovery**: When returning from interruption, the UI must surface the last session, run, and suggested next action immediately.

## Aesthetic Standards
-   **Muted Surfaces**: Avoid "neon chaos." Use a calm, professional palette.
-   **High Contrast**: Ensure text is easily legible under various conditions.
-   **Guided Motion**: Use animation only for attention guidance, never for decoration.
